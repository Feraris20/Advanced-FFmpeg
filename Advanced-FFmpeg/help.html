Advanced-FFmpeg
This processor builds and runs an FFmpeg command from the UI settings. It is currently focused on MP4 output and can encode video, encode or copy audio, scale video, add an image watermark, burn in timecode, and pass the rendered file to the next FFAStrans node as the new source.

FFmpeg Engine
Internal FFmpeg uses the FFAStrans FFmpeg variables.
External FFmpeg accepts a full path to ffmpeg.exe, an FFmpeg folder, or an FFAStrans variable such as %s_var%.
FFAStrans variables are expanded during execution before the processor validates the path. ffprobe.exe is expected next to ffmpeg.exe.

Container Format
MP4 is the first-release target and is enabled.
Other containers are visible as future options but disabled for now.

Video Codec
Choose the video encoder for the MP4 file. Hardware encoders require matching GPU support on the execution machine.
Copy does not re-encode video. The source video must already be valid for MP4. Filters such as Scale, Watermark, and Timecode Burn-in require re-encoding and are ignored with Copy.

Rate Control
ABR uses the Video Bitrate field.
2-Pass ABR runs an analysis pass to NUL and a second pass to the final MP4.
CRF uses constant quality. Lower values are higher quality and larger files.
QP/CQ uses FFmpeg -qp for constant quantizer mode.
Preset, Profile, Level, and Tune are passed to encoders when selected. Defaults do not add extra command options.

Framerate and GOP
Framerate can keep the source timing, use a preset, or use Custom.
Custom Framerate accepts values such as 25, 23.976, 24000/1001, or an FFAStrans variable such as %s_var%.
Keyframe Interval writes -g when a number or variable is supplied. Empty or 0 keeps FFmpeg default behavior.
Min GOP Size writes -keyint_min when a number or variable is supplied. Empty or 0 keeps FFmpeg default behavior.

Color and Display
Color Mode is rebuilt for the selected codec. Some codecs require a defined pixel format, so Keep Original may be hidden.
Display Mode can keep progressive output or add interlace flags for top-field-first or bottom-field-first encoding.
Frame-Packing is shown only for compatible x264/x265 software encoders.

Scale
Scale can keep the original size, use a preset size, or use Custom.
Custom Width and Custom Height accept positive numbers, -1, blank values, or FFAStrans variables.
If only one custom dimension is provided, the other side becomes -1 so FFmpeg keeps the aspect ratio.

Watermark
Enable Watermark adds an image watermark as a second FFmpeg input and uses filter_complex.
Image path accepts common FFmpeg image files such as PNG, JPG, BMP, TIFF, TGA, GIF, WEBP, or an FFAStrans variable path such as %s_var%.
Position uses a 3x3 grid, matching the timecode position picker.
X Offset and Y Offset accept numbers or FFAStrans variables and move the watermark from the selected grid point.
Resize can keep the original watermark size, fit inside the source, fit source width, or fit source height. The source reference is the video after Scale has been applied.
Opacity controls the watermark alpha from 0 to 100.
Watermark requires video re-encoding and is ignored when Video Codec is Copy.

Timecode Burn-in
Enable Timecode adds an FFmpeg drawtext timecode overlay.
Start Timecode is used when original source timecode is not enabled, or when the source fallback is set to use this field. It accepts timecode values such as 00:00:00:00 or 00:00:00;00. Empty values reset to 00:00:00:00.
Use original media start timecode makes the processor probe the source during execution with ffprobe and replace the command placeholder with the detected source timecode.
If no source timecode appears only when original source timecode is enabled. It can start from 00:00:00:00 or use the Start Timecode value above.
Timecode FPS can use the selected/source framerate or a fixed value.
Timecode Font can use FFmpeg default font handling, one of the common Windows font files, or a custom installed font name/full font path. The built-in list uses Arial, Calibri, Consolas, Courier New, and Tahoma.
Position chooses the burn-in location.
Timecode Size chooses a preset size or a custom percent value adjusted by the - and + buttons.
Background opacity controls the black box behind the timecode.
Foreground opacity controls the white timecode text opacity.

Audio
Audio Codec can encode with MP4-compatible codecs, copy the source audio, or disable audio with No Audio.
Copy writes -c:a copy and hides encode-only controls.
No Audio writes -an and hides audio encode controls.
Sample Rate, Bitrate, and Channels are used only when audio is encoded.
Track Name writes the audio stream title metadata and accepts FFAStrans variables.
Audio Delay accepts numbers or FFAStrans variables. Empty or 0 keeps default timing.
Language writes the three-letter audio language tag.

Advanced
Threads accepts a number or FFAStrans variable. Empty or 0 keeps FFmpeg default behavior.
Hardware Acceleration adds -hwaccel before the source input when selected.
Open GOP is Default or On and is shown only for compatible x264/x265 software encoders. Default adds nothing.
Faststart is Default or On for MP4. On adds -movflags +faststart.
Brand adds MP4 -brand only when a non-default brand is selected.
Codec ID adds -tag:v only when a supported non-default video tag is selected.
Force CFR is Default or On. On adds -vsync cfr.
Extra Arguments are appended to the FFmpeg command. Use this only for options that are valid for the selected codec and container.

Options
Generated Command is hidden by default. Set it to Shown when you want to inspect the exact command the UI is building.
Preset JSON exports the current settings to a JSON file and imports compatible JSON presets. The preview box fits inside the Options tab.
Presets include presetSchemaVersion so future incompatible formats can be rejected safely.

Execution Output
The processor sets s_source to the encoded output file. If an error occurs, it sets s_job_error_msg and exits with an FFAStrans error.
